Buttons

Colors

We have a number of button colors available that are used throughout our designs. Semantically, the class names are defined by action and not color.

<button class="lf-btn lf-btn-default" disabled="disabled">Disabled</button>
<button class="lf-btn lf-btn-default">Default</button>
<button class="lf-btn lf-btn-primary">Primary</button>
<button class="lf-btn lf-btn-warning">Warning</button>
<button class="lf-btn lf-btn-success">Success</button>
<button class="lf-btn lf-btn-info">Info</button>
<button class="lf-btn lf-btn-link">Link</button>

Sizes

There are also 3 additional sizes as well as a full-width block-level button size. These are designed to match input and form control heights.

<button class="lf-btn lf-btn-default lf-btn-lg lf-btn-block">Cancel</button>

Badges

Badges are used to denote counts of items. They can also be used as links. To de-emphasize a badge, append the modifier class, .lf-badge-mute.

1 2 120 
<span class="lf-badge">1</span>
<span class="lf-badge lf-badge-mute">2</span>
<a href="#" class="lf-badge">120</a>

Alerts

Colors

Alerts come in three flavors: .lf-alert-success, .lf-alert-info, and .lf-alert-warning.

Dismissable alerts

If the alert is dismissable, add class .lf-alert-dismissable and include a <span class="lf-close [light]"/>. Depending on the color of the alert, use the default darker close via fycon-x as seen in the example below.

Success: Good job, everything looks good!

Tooltips

Usage

For performance reasons, tooltip() is not loaded by default. You must initialize them yourself. You can do this via jQuery:

data-attribute: $('[data-toggle="tooltip"]').tooltip();

class/id selector: $('.tooltip').tooltip();

This is a tooltip. You can make them appear in a number of ways. Also to the left and down below.
$('#element').tooltip(
  {
    animation: true, // boolean
    html: false, // boolean; allows HTML to be inserted into the tooltip
    placement: 'top|bottom|left|right|auto', 
    trigger: 'click|hover|focus',
    title: 'string' // value if title attribute is not present on the element
    delay: int, // delay animation in ms
    container: 'string', // appends tooltip to a specific element
  }
);

Methods/Options

There are also a number of placement options to pass to tooltips. To access these, either configure them via the HTML5 Data API or in a javascript config:

data-attribute: <span data-toggle="tooltip" data-placement="top|bottom|left|right|auto">

Popovers

Usage

Popovers are syntatically similar to tooltips. In order to use popovers, you must include the tooltip.js lib as well as it is a dependency. Again, you are responsibile for initializing popovers yourself, and can access them via data-attribute or a selector.

<button class='lf-btn lf-btn-primary lf-btn-lg'
    data-toggle="popover" 
    data-placement="top" 
    data-content="Celebrate Mark Doten Day when the weather is hotter than 90°F!">
  Click me!
</button>

Usage

Include a template at the bottom of the page with class fade to hide it by default. Assign an id to this modal template and assign it as the data-target.

<button class="lf-btn lf-btn-lg lf-btn-default" data-toggle="modal" data-target="#lfModal">
  Launch Modal
</button>

Size options

You can also pass two additional modal sizes in lieu of the default 600px width on lf-modal-dialog:

  • .lf-modal-sm: 300px
  • .lf-modal: 600px
  • .lf-modal-md: 900px

Loader

Usage

Livefyre's loader depends on loader.js as a dependency (included in the minified js).

Options

loader accepts two parameters: elem and size (optional).

  • elem expects an object or jQuery object
  • size expects an integer divisible by 3 or one of three pre-canned sizes as a string: 'mini', 'small', or 'large'.

<-- HTML -->
<div id="lfLoader"></div>

<-- JS -->
loader.decorate($('#lfLoader'), 120)
loader.decorate($('#lfLoader-large'), 'large')
loader.decorate($('#lfLoader-small'), 'small')
loader.decorate($('#lfLoader-mini'), 'mini')
