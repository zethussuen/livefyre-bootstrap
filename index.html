Why Livefyre Bootstrap? v1.4.1

Use me and your designer will love you on your next project

Purpose of this project

In an effort to streamline designs across an ever expanding suite of enterprise products, the cool kids in design drafted a UI guide. What resulted was a need for a core set of coded and reusable components that we could pass over to our bff's in engineering. Hate coding that same button/dropdown/menu everytime you start on a new project? You've come to the right place.

Usage

Including livefyre-bootstrap in your next project is ezpz. In your bower.json, simply include the component and give yourself a $ bower install and you're almost there:

"livefyre-bootstrap": "git@github.com:Livefyre/livefyre-bootstrap.git#~1.4.1"
It's advised to pin to a certain version of livefyre-bootstrap in case of non-backwards compatible updates.

Once you've included and installed the component via bower, attach the class .lf to your containing element (usually <html class="lf">, or <body class="lf">).

Contributing

The livefyre-bootstrap repository sits on github and currently follows semver (sorta). If you find a bug or have a component request, please open an issue with any relevant information, markup, or screenshots.

Do NOT push directly to master. Please create a feature-branch and submit a pull request. Jenkins watches master for builds, generates release tags, and builds gh-pages. Zethus Phil will cry when you pollute master. He will also ignore all future feature requests from violaters of this crime.
